## 5. Retrieval Pipeline Architecture

The core of the project is the retrieval pipeline. Five independent and composable retrieval strategies have been designed and implemented to address different search modalities and edge cases:

### 5.1. Hybrid Retrieval
This module combines dense semantic search (ChromaDB embeddings using cosine similarity) with sparse lexical search (BM25 applied over the top 200 semantic candidates). The distinct scoring arrays are merged using Reciprocal Rank Fusion (RRF) with empirically determined weighting parameters (0.7 semantic / 0.3 lexical). Title-based deduplication is enforced to guarantee intra-list diversity.

### 5.2. Self-Querying
Leverages `Llama-3.3-70B` to translate natural language queries into structured JSON filters mapped to predefined metadata attributes (genre, director, year, Tomatometer, runtime), executing them alongside a base semantic search. Reliability and hallucination mitigation are enforced through three constraints:
* An explicit `AttributeInfo` schema definition.
* Few-shot prompting (4 highly specific English examples).
* A `temperature=0` setting to guarantee deterministic JSON generation.

### 5.3. Multi-Query
Utilizes `Llama-3.1-8B` to generate 5 semantic reformulations of the original query (in English, `temperature=0`). These variants are executed in parallel against the ChromaDB vector store. The retrieved subsets are unified and deduplicated, yielding up to 50 unique candidate reviews. This strategy significantly expands the latent space and proves highly effective for subjective, atmospheric, or abstract conceptual queries.

### 5.4. Parent Retrieval
Instead of returning isolated text chunks, this module retrieves individual reviews and aggregates them at the entity level (movie) utilizing the `link` metadata attribute as a foreign key. It dynamically constructs a consolidated "parent" document containing up to the 8 most relevant reviews for a single title. This provides a highly enriched context window for the LLM and achieved the maximum corpus coverage during testing (120 unique titles).

### 5.5. Combined Pipeline (Experimental)
Integrates the Self-Querying and Multi-Query architectures into a unified sequential pipeline. The structured filters generated by the Self-Query module dynamically truncate the global search space, while the Multi-Query reformulations perform deep semantic exploration exclusively within that restricted subset. Final candidate ranking is resolved via RRF. This experimental configuration is designed to maximize recall while strictly adhering to hard logical constraints.

## Despliegue del Entorno de Ejecución

La arquitectura del sistema requiere la ejecución concurrente de dos servicios independientes: el backend (API REST) y el frontend (Dashboard Analítico). Para inicializar la plataforma, es necesario operar sobre dos sesiones de terminal distintas en la raíz del proyecto.

### 1. Inicialización del Backend (API REST)
En la primera terminal, asegúrese de tener el entorno virtual activado. Ejecute el servidor Uvicorn para levantar el servicio de FastAPI. El proceso quedará en ejecución continua, escuchando las peticiones en el puerto 8001:
